A. 全模型 initializer dedup

python -m src.deduplicate_initializers --model-dir outputs\onnx --output-dir outputs\onnx_dedup_all --report outputs\reports\deduplicate_initializers_all.csv

然后只挑：

cost 降低明显

forbidden op 通过

结构没有语义变化

的任务生成单任务 ablation。

这是目前最值得继续做的低风险方向，因为它“不改变节点拓扑和 tensor 值”，只是合并 byte-identical initializer；你已经在 task209 / task157 / task233 上验证过这个思路。

B. 常量折叠 / 未使用 initializer 删除

优先做图等价 pass：

remove unused initializer
dedup identical constants
canonicalize int64/int32 indices
bool mask compression
shared Slice / Gather constant table

这类优化不应该改变任务语义，适合先做全量扫描。但仍然建议按任务生成 ablation，不要一次全合。

4. 高风险调整可以开始，但必须隔离

可以开始高风险规则研发，但不要把它叫“final 优化”，而应该叫：

high-risk ablation candidate generation

执行原则：

1. 新规则只生成 candidate，不直接 --replace。
2. 每个 zip 只替换一个 task。
3. online 分数 >= 当前 final 才合入。
4. online 分数 < 当前 final，立刻 reject，不讨论本地 cost。
5. 所有 rejected task 记录到 denylist，避免反复误合入。

你已经有 src.build_ablation_submissions，而且之前就是靠这个流程筛出 task025 / task084 / task200，同时挡住了 task028 / task367 / task396。

6. 风险分级建议

你可以按这个标准判断是否开始做：

类型	是否直接合入 final	说明
图等价优化，如 initializer dedup	可以先本地合入，但最好仍 ablation	不改语义，风险最低
已有线上正收益同类规则的小修	可以 ablation 后合入	例如 task084/task200 风格
新语义规则，train/test/arc-gen 全过	不直接合入	必须单任务 online ablation
component selection / object reasoning / pattern paste	高风险，只做实验	很容易本地过、线上掉
多任务批量替换	不建议	之前 5909 回退就是教训

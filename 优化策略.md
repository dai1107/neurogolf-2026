2. 优化 PanelSemanticRule：做 dynamic quadrant selector

你之前的 panel 规则主要是：

PanelSeparatorBinaryOpRule: 7 solved
PanelSemanticRule: probe 有 2 个剩余候选

但现在 PanelSemanticRule 还没有真正变成强 builder。你应该让 Codex 专门实现：

DynamicQuadrantPanelSelectBuilder

它针对 2x2 panel 输入，不要求固定选择第几个 panel，而是根据 panel 特征动态选择：

selector = contains_unique_color
selector = color_absent_from_other_panels
selector = most_non_background
selector = least_non_background
selector = different_from_other_panels
selector = matches_color_role

ONNX 实现思路：

1. 将输入切成 4 个 quadrant/panel。
2. 对每个 panel 统计：
   - 每种颜色出现次数
   - 非背景面积
   - 是否含唯一颜色
3. 得到 4 个 selector score。
4. 用 OneHot / Equal / Where / Mul / Add 选择目标 panel。
5. 输出选中的 panel，可选再接 color map / mirror / rotate。

这类 builder 不要做太泛。先只支持：

2x2 等尺寸 panel
1x3 / 3x1 等尺寸 panel
输出等于选中 panel
输出等于选中 panel + color_map
输出等于选中 panel + mirror/rotate

预期收益：+2 到 +5。如果 task065/task207 能吃掉，这个方向就值得继续扩。

3. 优化 FrameInteriorRule：从 probe-only 变成正式 builder

当前剩余候选里 FrameInteriorRule 有 5 条，是最值得转 builder 的现有规则。你已经有 DynamicNonBackgroundBBoxCropRule，说明动态 bbox 技术已经跑通，并且用了 ArgMax / Gather / ReduceSum 这类算子生成动态 bbox crop，已经通过本地 ONNX checker、onnxruntime 和 submission 检查。

现在可以复用这个思路做：

DynamicFrameInteriorCropBuilder

支持：

output = frame interior crop
output = frame interior crop + color map
output = frame interior fill
output = remove frame, keep inside
output = keep frame, erase outside

匹配逻辑：

1. 枚举 frame_color。
2. 判断该颜色是否构成矩形边框。
3. 动态求 frame 的 top / bottom / left / right。
4. interior = [top+1:bottom, left+1:right]。
5. 验证 output 是否等于 interior 或 interior+color_map。

ONNX builder 初版先做：

color-specific frame bbox
frame interior crop
frame interior crop + color map

不要一开始做复杂“多个 frame 选择”。先吃掉剩余 5 个 candidate 中最简单的。

预期收益：+3 到 +5。

4. 优化 DynamicBBoxCropRule：从 non-background bbox 扩到 color-specific bbox

现在正式 builder 是：

DynamicNonBackgroundBBoxCropRule

它解决了 task031, task259, task177。

下一步要扩为：

DynamicColorBBoxCropRule

支持：

bbox of a specific color
bbox of rarest non-background color
bbox of most frequent non-background color
bbox of unique color
bbox of color absent/present under marker relation
bbox crop + color map
bbox crop + mirror/rotate

关键不是“任意 component”，因为 connected component 动态 ONNX 很难；先以 color mask bbox 作为安全替代。

可选 selector：

selected_color = unique non-background color
selected_color = rarest non-background color
selected_color = most frequent non-background color
selected_color = color not touching border
selected_color = color inside frame
selected_color = marker color / marker-neighbor color

ONNX 实现思路：

1. 对每个颜色 channel 计算 color_count。
2. 用 ReduceSum 得到每个颜色面积。
3. 用 ArgMax / ArgMin / Equal 选出目标颜色。
4. 构造 selected_color_mask。
5. 对 selected_color_mask 求 bbox。
6. 动态 Gather crop 到左上角。

预期收益：+4 到 +10。

这个是从 58 往 65 推的关键。
